## Checklist Google Shopping: Antes de Gastar Um Real em Anúncios

Se você vai investir R$100/dia em Google Shopping, antes de ligar a campanha, precisa validar seu catálogo.

Catálogo ruim = anúncios pausados, cliques baixos, conversão zero.

Este artigo é um **checklist de 50 itens** que você pode imprimir, preencher, e garantir que seu catálogo está pronto para vender.

---

## Como Usar Este Checklist

1. **Imprima ou abra em PDF** (veja link ao final)
2. **Escolha 10-20 produtos aleatórios** do seu catálogo
3. **Verifique cada item abaixo** para esses produtos
4. **Se 90%+ passam em cada categoria**, seu catálogo está OK
5. **Se <90%**, corrija problema antes de ativar campanha

---

## Seção 1: Campos Obrigatórios (5 Itens)

```
☐ ID único (SKU) — cada produto tem ID diferente, sem duplicatas
☐ Title — título preenchido, não vazio
☐ Price — preço é número válido (não "gratuito", não R$0)
☐ Image Link — URL da imagem válida (começa com http://)
☐ Availability — "in stock" ou "out of stock", não vazio
```

---

## Seção 2: Qualidade de Imagem (5 Itens)

```
☐ Imagem carrega (teste clicar URL, deve abrir imagem)
☐ Tamanho mínimo 1000x1000px (maior é melhor, até 5000x5000)
☐ Fundo branco ou neutro (não colorido, não gradiente)
☐ Sem marca d'água, sem logo, sem texto sobreimposto
☐ Produto destacado (não pequeno, não cortado, visível 100%)
```

---

## Seção 3: Estrutura de Dados (10 Itens)

```
☐ Encoding UTF-8 (se XML, começa com <?xml version="1.0" encoding="UTF-8"?>)
☐ Tags XML bem formatadas (se XML, todas fechadas corretamente)
☐ Sem caracteres especiais quebrados (acentos aparecem corretos, não "?")
☐ Preço com currency (ex: "49,90 BRL", não só "49,90")
☐ Moeda é BRL (Brasil), não USD ou outra
☐ Formato de data ISO (se tiver data, use YYYY-MM-DD)
☐ Links válidos (não retornam 404)
☐ Sem espaços extra (trimmed — sem espaço no início/fim)
☐ Sem line breaks desnecessários (quebras de linha só onde devem)
☐ Arquivo XML valida (use validador online: xmlvalidation.com)
```

---

## Seção 4: Conteúdo de Título (5 Itens)

```
☐ Título é específico (não genérico como "Produto", "Camiseta")
☐ Título tem 5+ palavras (mínimo descritivo)
☐ Título tem <150 caracteres (máximo Google)
☐ Contém marca, tipo, cor (essencial para matching)
☐ Sem números aleatórios, sem pontuação excessiva
```

---

## Seção 5: Descrição (3 Itens)

```
☐ Descrição preenchida (não vazia)
☐ Descrição tem 50+ caracteres (não muito curta)
☐ Descrição descreve produto (não é título repetido)
```

---

## Seção 6: Atributos Recomendados (8 Itens)

```
☐ Brand — marca do produto
☐ Color — cor principal (ex: "Azul", não "cor azul")
☐ Size — tamanho (ex: "P", "M", "G", ou "PP-M-G")
☐ Material — material principal (ex: "100% algodão", "Poliéster")
☐ Gender — gênero (masculino, feminino, unissex)
☐ Product Type — categoria (ex: "Vestuário > Camisetas > Básicas")
☐ Item Group ID — para agrupar variações (mesma camiseta, cores diferentes)
☐ GTIN — código de barras (se tiver)
```

---

## Seção 7: Preço e Promoção (4 Itens)

```
☐ Preço é consistente com website (diferença <2%)
☐ Sale price (se em promoção) é menor que preço normal
☐ Ambos preços têm currency (ex: "R$")
☐ Preço não é 0, não é negativo
```

---

## Seção 8: Disponibilidade (3 Itens)

```
☐ Availability é "in stock" ou "out of stock" (não outro valor)
☐ Status combina com estoque real (produto com 0 = out of stock)
☐ Disponibilidade atualiza quando estoque muda (<4h de delay)
```

---

## Seção 9: Imagens Adicionais (2 Itens)

```
☐ Additional image links — pelo menos 3 imagens (5 é ideal)
☐ Todas as imagens carregam (nenhuma é 404)
```

---

## Seção 10: Validação Google Merchant Center (3 Itens)

```
☐ 0 erros críticos (produtos não pausados por erro)
☐ <5% avisos amarelos (problemas menores, toleráveis)
☐ 100% de produtos foram indexados (não perdeu nenhum)
```

---

## Seção 11: Compliance Brasil (4 Itens)

```
☐ Política de devolução clara (7+ dias conforme Lei)
☐ Endereço da loja real e verificável
☐ Telefone/email válido e responsivo
☐ Sem produto proibido (fake, arma, medicamento sem prescrição)
```

## Erros Mais Comuns Que Aparecem No Checklist

### Erro 1: Imagem 404

Solução: substitua URL de imagem no feed.

Como testar: copie URL, abra no navegador. Deve carregar imagem. Se 404 → erro.

### Erro 2: Encoding Errado (Acentos quebrados)

Solução: converta arquivo para UTF-8 (use Sublime Text ou iconv).

Como testar: procure por "ç" ou "ã" no arquivo. Se aparecem como "?" → encoding errado.

### Erro 3: Preço Inconsistente Feed vs Website

Solução: sincronize automaticamente (a cada 1-2h) em vez de manual.

Como testar: escolha 5 produtos. Veja preço no feed. Veja preço no website. Devem ser iguais.

### Erro 4: Titulo Genérico

Solução: use fórmula [MARCA][TIPO][COR][TAMANHO][MATERIAL].

Errado: "Camiseta"
Certo: "Camiseta Azul Básica Feminina 100% Algodão Tamanho P"

### Erro 5: Item Group ID Não Agrupa Corretamente

Solução: todos as variações (cores/tamanhos) do mesmo produto devem ter mesmo item_group_id.

Exemplo:
```
CAMISA-001-P-AZUL: item_group_id = CAMISA-001
CAMISA-001-P-BRANCO: item_group_id = CAMISA-001
CAMISA-001-M-AZUL: item_group_id = CAMISA-001
```
