## Checklist Meta Ads: Valide Seu Catálogo Antes de Gastar

Meta Ads é mais forgivente que Google Shopping, mas ainda precisa de catálogo estruturado.

Catálogo ruim em Meta = produtos rejeitados, anúncios não aprovados, campanhas pausadas.

Este artigo é **checklist de 40 itens** para validar seu catálogo Meta antes de criar primeira campanha.

---

## Como Usar Este Checklist

1. **Escolha 10-15 produtos** aleatórios do seu catálogo
2. **Verifique cada item abaixo** para esses produtos
3. **Se 85%+ passam**, seu catálogo Meta está OK
4. **Se <85%**, corrija antes de ativar campanhas

---

## Seção 1: Campos Obrigatórios (5 Itens)

```
☐ ID único — cada produto tem ID diferente
☐ Title — título preenchido
☐ Price — preço válido (número, não texto)
☐ Image URL — imagem carrega, não é 404
☐ Link — URL do produto no seu site válida
☐ Availability — "in stock" ou "out of stock"
```

---

## Seção 2: Qualidade de Imagem (5 Itens)

```
☐ Imagem carrega (testa URL no navegador)
☐ Tamanho mínimo 500x500px (Meta não rejeita, mas melhor 1000x+)
☐ Proporção recomendada (1:1 quadrada ideal, 4:5 aceita)
☐ Imagem clara (sem blur, sem distorção)
☐ Sem marca d'água ou watermark visível
```

**Diferença vs Google:** Meta é mais flexível. Aceita imagem menor e estilos diferentes.

---

## Seção 3: Título e Descrição (5 Itens)

```
☐ Título <100 caracteres (Meta trunca após)
☐ Título descritivo (não genérico como "Produto")
☐ Descrição preenchida (importante para Meta, mais que Google)
☐ Descrição <500 caracteres (recomendado)
☐ Descrição em tom amigável (não muito técnico, Meta é social)
```

**Diferença vs Google:** Meta adora descrição emocional, narrativa. Google quer técnica.

---

## Seção 4: Preço e Promoção (5 Itens)

```
☐ Price preenchido (número válido)
☐ Currency declarado (BRL ou R$)
☐ Sale price (se em promoção) < price normal
☐ Sale price mostra desconto real (ex: de R$100 por R$79, não invertido)
☐ Preço consistente com website (diferença <5%)
```

**Meta adora sale_price:** mostra "de R$100 por R$79" — visual de oferta.

---

## Seção 5: Categorização (3 Itens)

```
☐ Category preenchida (ajuda Meta organizar catálogo)
☐ Category é simplificada (ex: "Camisetas", não "Vestuário > Roupas > Camisetas > Básicas")
☐ Valor de category é padrão Meta (usa terminologia correta)
```

**Diferença vs Google:** Meta quer category simples (1 nível). Google quer hierarquia (3-5 níveis).

---

## Seção 6: Atributos Complementares (3 Itens)

```
☐ Brand — marca do produto
☐ Color — cor (se relevante para produto)
☐ Size — tamanho (se produto tem tamanhos)
```

**Meta não exige**, mas recomendado para filtros melhor em anúncios.

---

## Seção 7: Variações (Item Group ID) (2 Itens)

```
☐ Produtos com variações (cores, tamanhos) têm item_group_id
☐ Item group ID é igual para mesma "família" (ex: camiseta azul P, M, G = mesmo group)
```

**Meta necessário:** para mostrar variações em carrossel.

---

## Seção 8: Links e URLs (3 Itens)

```
☐ Link aponta ao produto específico (não homepage)
☐ Link carrega sem erro (não 404)
☐ Link é HTTPS (seguro, recomendado)
```

**Crítico em Meta:** link quebrado = campanha pausada.

---

## Seção 9: Conformidade e Política (4 Itens)

```
☐ Nenhum produto proibido (fake, arma, medicamento)
☐ Produto não viola direitos autorais (imagem própria ou com permissão)
☐ Descrição não faz claim falso (não diz "cura" se medicamento, etc)
☐ Preço não engana (não "de R$100" quando sempre foi R$79)
```

**Meta é rígido:** viola política = catálogo inteiro pode ser deletado.

---

## Seção 10: Dados Técnicos (3 Itens)

```
☐ CSV/XML bem formatado (sem erros de parsing)
☐ Encoding UTF-8 (acentos aparecem corretos)
☐ Sem valores duplicados ou conflitantes
```

---

## Seção 11: Status e Estoque (3 Itens)

```
☐ Availability é "in stock" ou "out of stock"
☐ Status combina com estoque real (0 = out of stock)
☐ Disponibilidade atualiza quando muda (ideal <24h)
```

---

## Seção 12: Validação Meta (2 Itens)

```
☐ Catálogo importado em Meta Business Manager
☐ 0 erros críticos (produtos não rejeitados)
```

---

## Pontuação: Está Pronto?

Se você verificou 10-15 produtos aleatórios:

- **Passaram 95-100%:** ✓ Perfeito, ativa campanhas
- **Passaram 85-94%:** ✓ Bom, ativa e monitora
- **Passaram 70-84%:** ⚠️ Precisa ajustes
- **Passaram <70%:** ✗ Não ativa, corrija primeiro

## Erros Mais Comuns em Meta

### Erro 1: Sale Price Confundindo

Errado: price = R$79, sale_price = R$100 (parece desconto ao contrário)

Certo: price = R$100, sale_price = R$79 (desconto real)

Meta mostra: "de R$100 por R$79" — visual correto.

### Erro 2: Category Muito Específica

Errado: "Vestuário > Roupas Femininas > Camisetas > Básicas > Brancas > Algodão"

Certo: "Camisetas" ou "Roupas Femininas > Camisetas"

Meta quer 1-2 níveis, não 6.

### Erro 3: Link Aponta Homepage

Errado: link = https://loja.com (homepage)

Certo: link = https://loja.com/produtos/camiseta-azul-p-001 (específico)

Cliente clica anúncio, deve chegar direto no produto.

### Erro 4: Descrição Muito Técnica

Errado: "100% algodão Egípcio, tessitura 200 fios, peso 180g/m²"

Certo: "Camiseta confortável perfeita para o dia a dia, super macia e durável"

Meta é social. Cliente quer imaginar a si mesmo usando, não specs técnicas.

### Erro 5: Imagem com Muita Composição

Errado: Imagem de modelo usando 3 roupas diferentes (qual é o produto?)

Certo: Imagem clara do 1 produto, pode ser isolado ou modelo usando

Meta quer deixar claro qual é o produto.

## Caso Real: Loja Que Recuperou Catálogo Meta Pausado

Uma loja em Salvador tinha catálogo Meta rejeitado. Meta simplesmente deletou toda a campanha (política severa).

**Problema encontrado (após análise):**

```
❌ 150 produtos tinham "sale_price" > "price" (desconto ao contrário)
❌ 80 produtos tinham imagem de 200x200px (muito pequena)
❌ 200 produtos com title genérico ("Produto", "Item")
❌ Descrição vazia em 300 produtos
❌ 5 produtos eram falsificados (Meta detectou e pausou tudo)
```

**Ações:**

1. Removeu produtos falsificados (5 produtos)
2. Corrigiu sale_price vs price (150 produtos)
3. Ampliou imagens (80 produtos, refotografou)
4. Otimizou títulos (200 produtos)
5. Preencheu descrições (300 produtos com descrição emocional)

**Resultado:**

- Resubmeteu catálogo
- Meta aprovação em 48h
- Catálogo agora com 950 produtos válidos (começou com 1000, tirou 50 ruins)
- Primeira campanha: 2.1x ROAS (ótimo)
